## 1. What this is
A cross-platform (Android now) **personal Quran companion**: read the Quran, hear
recitation, understand each ayah via **authentic Shia tafsir**, look up **word
meanings**, and use AI (voice or text) to navigate, search, and ask questions ‚Äî
with a futuristic galaxy UI. Owner: **Majid** (single user, not public). Default
language **Persian (fa)**; also en / nl. Arabic is the sacred text.

**Architecture now: the app is self-contained.** There is **no gateway**. All
Quran text, search, the reciter catalog and audio URLs are bundled in the APK and
served on-device. The **only** network calls are:
- **OpenAI directly** (AI features + Whisper speech-to-text), key baked in at build
  time via `--dart-define=OPENAI_API_KEY`;
- **EveryAyah CDN** for recitation MP3s (streamed/downloaded).

Two hard rules still hold: (1) Arabic text is byte-accurate from verified Tanzil,
never AI-generated; (2) the app executes only a **fixed intent allow-list**, never
a model-returned command string.

## 4. Feature deep-dives (this session)

### 4a. Standalone content (`core/local_content.dart`)
`LocalContent` loads `quran.json.gz` once into memory and exposes `listSurahs`,
`getSurah`, `search`, `reciters`, `audioUrl`, `surahAudioUrls`. Replaced the old
`ApiClient` (deleted). Providers: `localContentProvider`; `surahListProvider` /
`surahProvider` await `ensureLoaded()`. Audio URLs are built locally from the
EveryAyah scheme (`https://everyayah.com/data/{folder}/{SSS}{AAA}.mp3`).
- **Regenerating the bundled assets** (the gateway that produced them is gone): you
  must re-fetch the Tanzil editions (ar.simple, fa.makarem, en.qarai, nl.siregar)
  + EveryAyah reciter manifest and re-emit `quran.json.gz` / `reciters.json`.

### 4b. Full-text search (`features/search/`)
`local_content.search(query, lang, surah?)`: matches Arabic **and** the active
translation, word-AND any order, substring. `normalizeArabicForSearch()` strips
diacritics/tatweel and folds alef/ya/hamza/ta-marbuta + Persian yeh/kaf ‚Äî **define
the regex with `\u` escapes** (literal combining chars get byte-scrambled by
editors and silently break the ranges; this caused a real bug). `highlighted_text.dart`
holds the shared `HighlightedText` (animated glow on matched whole words; splits
text with `RegExp(r'\S+').allMatches` to PRESERVE whitespace ‚Äî Dart's `split` drops
delimiters) and `AyahResultCard`.

### 4c. Reader (`features/reader/surah_reader_page.dart`)
Now a `ScrollablePositionedList`. Params: `initialAyah` (open scrolled+highlighted),
`autoplayFrom` (start recitation), `autoOpenTafsir` (open the tafsir sheet for the
ayah). A **sparkle FAB** opens a surah-scoped "Ask about this surah" sheet
(`AskQuranSheet(surah:‚Ä¶, persist:false)`).

### 4d. Voice input (`features/ai/ai_input_bar.dart`)
Shared input bar (mic + field + send) used by both AI sheets and the home smart box
(`sheetStyle:false` embeds it). Flow: **record (`record` pkg, m4a) ‚Üí Whisper STT
(`whisper-1`, lang = translation language) ‚Üí context cleanup (`refineTranscript`) ‚Üí
drop into box ‚Üí auto-submit**. So the spoken question runs like a typed one and the
corrected transcript is shown as the asked question. Futuristic animation: pulsing
mic halo, live waveform while listening, shimmer while "Transcribing‚Ä¶/Polishing‚Ä¶".

### 4e. AI Q&A + scope handling (`features/ai/openai_client.dart`, sheets)
Two answer paths (decided by the real tafsir, **not** a guess):
- **`answer()`** = the *in-scope attempt* ‚Äî answers USING ONLY the provided ayah
  material; returns `in_scope=true` with the answer, or `in_scope=false` + a short
  "not covered" note (never uses outside knowledge).
- **`answerBeyond()`** = *out-of-scope* ‚Äî sends the **bare question only** (no
  ayah/tafsir) to the OOS model, with precise references required.
- Flow (home `ask_quran_sheet` and per-ayah `ai_summary_sheet`): locate ayat ‚Üí (if
  none ‚Üí out-of-scope) ‚Üí gather tafsir ‚Üí `answer()` attempt ‚Üí if `in_scope` show it,
  else resolve per the **out-of-tafsir mode** (see ¬ß4f). `summarize()` and
  `locateAyat()` unchanged. The router's `scope_guess` is no longer used.
- Home sessions: `ask_session.dart` (model, ‚â§5 turns, cumulative refs, history),
  `ask_sessions_controller.dart` (persisted to `ask_sessions.json` in app docs),
  `ask_sessions_sheet.dart` (new / continue / delete). Out-of-scope answers show a
  scope banner + references; in-scope show related-ayah chips.

### 4f. Settings (`features/settings/`, `core/config.dart`)
Four **model roles**, each its own picker, persisted (`flutter_secure_storage`),
seeded in `main.dart`, providers in `ai_settings_controller.dart`:
- `answerModelProvider` ‚Äî in-scope answers / summaries / locate.
- `oosAnswerModelProvider` ‚Äî out-of-scope (beyond-tafsir) answers.
- `classifyModelProvider` ‚Äî command routing.
- `refineModelProvider` ‚Äî voice transcript cleanup.
(Old single `ai_model` migrates into the answer model.) Plus
`outOfScopeModeProvider`: **Tafsir only** / **Answer with sources** (default) /
**Ask first** (confirm before a beyond-tafsir answer; confirmation lives in the
sheet, after the attempt reveals scope). Whisper STT model stays `whisper-1`.

### 4g. AI command assistant (`features/assistant/`)
The home search box is a single **auto-detect** bar (the old Surahs/Keywords toggle
is gone). On submit/voice:
1. **Local fast-path** ‚Äî a bare surah number/name navigates with **0 AI calls**.
2. Otherwise `OpenAiClient.routeCommand()` (Command model) returns strict JSON ‚Üí
   `RoutedCommand.fromJson` parses it against the allow-list (`app_command.dart`).
3. `CommandDispatcher` validates (surah 1‚Äì114, ayah in range via `LocalContent`)
   and executes. Live surah-name filtering still happens as you type.

Allow-list intents: `open_surah`, `recite` (autoplay), `search_quran` (inline
results), `search_in_surah`, `open_tafsir`, `ask` (‚Üí Ask sheet, auto-submitted),
`none`. Unknown/invalid ‚Üí `NoneCommand` (a hint toast). See `AI_ASSISTANT_PLAN.md`
for the full flowchart + per-route AI-call counts.

## 8. Decisions & caveats (don't re-litigate)
- **No gateway.** App is standalone; AI/STT go direct to OpenAI; audio from EveryAyah.
- **Default language Persian (fa).**
- **Scope is decided from the real tafsir** (in-scope attempt ‚Üí `in_scope`), not a
  pre-guess. Out-of-scope answers get the **bare question only** (no ayah/tafsir) +
  the **OOS model** + required references. Mode: tafsir-only / with-sources / ask-first.
- **Four AI model roles** are user-configurable; STT is `whisper-1`.
- **Allow-list only** for commands (hard rule #2). Surah/ayah always range-validated.
- **Arabic search regex must use `\u` escapes** (combining-char ranges get scrambled).
- **Voice questions auto-send** after a context-cleanup pass so the shown transcript
  reads correctly.
- Licensing to revisit before any public release: EveryAyah audio, Tanzil
  translations (non-commercial), Furqan tafsir (CC BY-ND), al-Tahqiq copyright.
- AI citations are **model-generated, not verified** ‚Äî verify before relying.
