## 1. What this is
A cross-platform (Android now; iOS/Web later) **personal Quran companion**:
read the Quran, hear recitation, understand each ayah via **authentic Shia
tafsir**, look up **word meanings** by tapping any Arabic word, with a
futuristic animated UI. Owner: **Majid** (single user, not public yet).
Languages: Arabic (sacred text) + fa / en / nl.

**Architecture:** Flutter app ⇄ (HTTPS+Bearer) ⇄ FastAPI gateway (owns secrets,
AI provider seam, OpenAI STT/TTS). Two hard rules: (1) Arabic text is
byte-accurate from a verified source, never AI-generated; (2) the app only
executes a fixed intent allow-list, never model-returned commands.

**Authenticity rule extends to content:** tafsir/lexicon must be authentic
human works, not AI-generated. (This is why AI-translated editions were
deliberately excluded — see §4, §5.)

## 2. How to run (verified recipe)
Prereqs already on this machine: Python 3.11, Flutter 3.44.2 (`~/flutter`),
Android SDK (`~/Android/Sdk`), adb, gateway venv at `gateway/.venv`.
Phone: Samsung Galaxy S23 Ultra, serial **R3CW50DLQFK**.

```bash
# 1) Gateway (token default dev-token-change-me; root .env holds OPENAI_API_KEY)
cd gateway && . .venv/bin/activate && \
  DEVICE_TOKEN=dev-token-change-me python -m uvicorn qis.main:app --port 8077

# 2) Forward phone → gateway
~/Android/Sdk/platform-tools/adb -s R3CW50DLQFK reverse tcp:8077 tcp:8077

# 3) Build + run on the phone
cd app && ~/flutter/bin/flutter run --debug -d R3CW50DLQFK \
  --dart-define=GATEWAY_URL=http://localhost:8077 \
  --dart-define=DEVICE_TOKEN=dev-token-change-me

# Tests
cd gateway && . .venv/bin/activate && python -m pytest -q   # 38 passing
cd app && ~/flutter/bin/flutter analyze lib                 # clean (4 pre-existing doc-comment infos)
```

**Gotchas**
- `flutter run` exiting **removes the `adb reverse`** — re-run step 2.
- The gateway is **not a daemon** and is **stateless across restarts**; if a
  *stale* instance is on :8077 (missing new routes → 404s), kill it and restart:
  `pid=$(ss -ltnp | grep :8077 | grep -oP 'pid=\K[0-9]+'); kill $pid`.
- New **native** plugins (sqflite, just_audio, secure_storage) require a full
  rebuild, not hot reload. New pure-Dart deps just need a re-run.
- To drive the UI headless for screenshots: `adb ... exec-out screencap -p > x.png`,
  `adb ... shell input tap X Y`, `adb ... shell input swipe …`, and
  `adb ... shell am force-stop com.imamshahid.quran_imam_shahid` then
  `monkey -p com.imamshahid.quran_imam_shahid -c android.intent.category.LAUNCHER 1`
  for a clean launch to the home screen.

## 4. Feature deep-dives (this session's work)

### 4a. Recitation audio
- **Source: EveryAyah** per-ayah MP3s, URL `https://everyayah.com/data/{folder}/{SSS}{AAA}.mp3`.
- Gateway: `content/reciters.json` (24 verified reciters, `default_reciter: parhizgar`,
  tagged shia/sunni/universal), `content/reciters.py` (`ReciterCatalog`),
  `routes/audio.py` → `GET /v1/reciters`, `/v1/audio/{reciter}/{surah}/{ayah}`,
  `/v1/audio/{reciter}/{surah}`.
- App: `features/audio/audio_controller.dart` (just_audio; single-ayah + continuous
  surah; local-file-if-downloaded else stream), `download_manager.dart` (per-surah
  offline cache in app docs), `player_bar.dart` (reciter picker grouped by tradition,
  prev/next, play, download w/ progress — now frosted glass).
- Reader auto-scrolls to keep the reciting ayah visible (GlobalKey + ensureVisible).
- License note: EveryAyah free/widely-used, no formal per-reciter license → fine for
  personal scope.

### 4b. Book tafsir (embedded, offline)
- **Source: [app-furqan/quran-app-data](https://github.com/app-furqan/quran-app-data)**
  (CC BY-ND 4.0, © Furqan Apps — attribution shown in-app, no modifications).
- **Only the Persian editions** are used (al-Mizan, Nemooneh, Noor) — they are
  authentic human works. The repo's **English/Urdu are AI-produced → excluded**
  (hard rule #1). They remain inside the archives (files unmodified for license)
  but the code reads only the Persian column.
- **Embedded in APK** (user choice over a smaller transformed form, for license
  cleanliness): `app/assets/tafsir/*.tar.xz` (~28 MB committed). On first use of an
  edition, `features/tafsir/tafsir_db.dart` extracts its `.db` (xz→tar in a
  background isolate via `compute`) to app docs, queries with **sqflite**. **No
  gateway needed for tafsir.** First open of each edition has a one-time extraction
  pause (al-Mizan/Noor ~40 MB, Nemooneh ~85 MB on device).
- Schema (each DB): `ayah_mapping(surah_number,ayah_number,content_id)` (all 6236),
  `content(content_id, <persian col>, …)` Markdown. Passage-based: one block can
  span many ayat (al-Mizan ≈10/block, Nemooneh ≈3.6, Noor ≈1.6 ≈ per-ayah). The
  sheet shows "covers ayat X–Y together" when a block spans >1 ayah.
- App: `reader/tafsir_sheet.dart` — source picker chips, Markdown via
  `flutter_markdown_plus`, **strips `<span>` anchors + ```arabic/```farsi
  info-strings** at render time (`_cleanMarkdown`), attribution footer.
- Gateway equivalents still exist (`content/tafsir.py`, `routes/tafsir_book.py`,
  `/v1/tafsirs`, `/v1/tafsir/{id}/{surah}[/{ayah}]`) but the **app no longer uses
  them** (kept for other clients/tests). Gateway extracts from committed `.tar.xz`;
  extracted `.db` are git-ignored (see `content/tafsir_data/SOURCES.md`).

### 4c. Word lexicon (tap-a-word → root → dictionary)
- **Mufradat (Raghib al-Isfahani)** — public domain. Scraped from **Shamela book
  23636** (881 pages), segmented by `[root]` markers (`<span class="c4">`) → 1,425
  root entries. Scripts in `tools/lexicon/` (`scrape_mufradat.py`,
  `build_wordroot.py`, `build_lexicon.py`).
- **Word→root** from the Quranic morphology corpus
  ([mustafa0x/quran-morphology](https://github.com/mustafa0x/quran-morphology)) →
  a global `normalized-form → root` map (~74% of Tanzil tokens; the rest are
  rootless particles, correctly no entry). Mufradat roots matched to corpus roots
  (~86% coverage) with weak/hollow/hamza/geminate canonicalization.
- Bundled as **`app/assets/lexicon/lexicon.db.gz`** (1.2 MB) → gunzip on first run →
  sqflite. Tables: `word_root(form→root)`, `entry(book,root,content)`.
- App: `features/lexicon/lexicon_db.dart` (engine + `normalizeArabic`),
  `lexicon_sheet.dart` (shows word, root, entries), tappable words in
  `reader/surah_reader_page.dart` (`_TappableArabic`).
- ⚠️ **CRITICAL parity:** Dart `normalizeArabic()` MUST stay byte-identical to the
  Python `norm()` in `tools/lexicon/build_wordroot.py` (it builds `word_root.form`).
  If you change one, change both and rebuild `lexicon.db.gz`.
- **al-Tahqiq (Mostafavi) — NOT integrated.** Contemporary & **copyrighted**; no
  clean web text exists (Shamela lacks it, archive.org OCR is garbled). User chose
  to **provide/license** a clean copy. Drop-in is ready:
  `python3 tools/lexicon/add_book.py tahqiq "al-Tahqiq" "التحقيق" "Hasan Mostafavi" <file>`
  (accepts root-keyed JSON or TSV), then uncomment the `'tahqiq'` `LexiconBook` in
  `lexicon_db.dart` and rebuild. See `tools/lexicon/README.md`.

### 4d. UI redesign — multi-theme + galaxy cosmic-zoom
- **Theming:** `core/theme.dart` (`AppPreset` + 5 presets: Sahar/Aurora/Emerald/
  Amber/Plum; shared rounded/elegant design language + fade-through page transition),
  `core/theme_controller.dart` (selection + light/dark/auto, persisted in secure
  storage, loaded **before first frame** in `main.dart`; theme changes cross-fade).
  Theme picker: `features/settings/theme_picker_sheet.dart`.
- **Shared widgets:** `widgets/app_card.dart` (gradient-border glass card, accent
  glow when selected, `translucent` variant — **no blur**, safe in lists),
  `widgets/glass.dart` (real `BackdropFilter` glass — used ONLY on fixed surfaces:
  app bars, player bar), `widgets/app_background.dart` (aurora gradient — used by
  the **reader**).
- **Home = galaxy cosmic-zoom** (`widgets/galaxy_background.dart`):
  - A **fixed world, moving camera**. Scroll = **zoom-out through cosmic scales**:
    Solar System → Milky Way → distant galaxies (layers cross-fade & shrink toward a
    focal point). **Idle = no camera drift** — only planets orbit, the galaxy
    rotates, stars twinkle.
  - **Real Solar System:** the 8 actual planets with correct colours / relative
    sizes / inner-faster orbits, gas-giant **banding** (Jupiter/Saturn), **Saturn's
    rings** (front/back split), Earth continents, a glowing pulsing sun.
  - **Milky Way:** rotating 2-arm log-spiral starfield + bulge + disk glow.
  - **Distant galaxies:** ~22 tilted glowing disks.
  - Driven by a monotonic `Ticker` (orbits/rotation/twinkle) + the list's `scroll`
    (a `ValueNotifier` wired from the home `ScrollController`). Zoom level
    `z = scroll / (height·2.4)`. Home renders in the preset's **dark** theme
    (space scene) with `AppCard(translucent: true)` cards floating over it.
  - **Performance:** all cheap canvas fills (no blur in the galaxy), stars/MW/
    galaxies precomputed once (seeded), whole scene in a `RepaintBoundary`. Note it
    repaints every frame while visible (continuous animation, by design) — fine on
    the S23; if a low-end device struggles, lower star/MW counts or gate the Ticker.

## 8. Decisions & caveats (don't re-litigate)
- **Tafsir/lexicon embedded in the APK** (not gateway-served) for true offline.
  Tafsir from `.tar.xz` (extract on first use); lexicon from `lexicon.db.gz`.
- **Persian-only** authentic tafsir; **Mufradat (Arabic)** for the lexicon. AI
  editions excluded by authenticity rule.
- **al-Tahqiq is copyrighted** → only via a user-provided licensed copy.
- **Licensing to revisit before any public release:** EveryAyah audio, Tanzil
  translations (non-commercial), Furqan tafsir (CC BY-ND — no derivatives), and
  al-Tahqiq copyright.
- **Dart/Python normalization parity** for the lexicon (see §4c) — fragile, keep
  in sync.
- **Galaxy home is always a dark space scene** (renders in the preset's dark theme)
  regardless of the app's light/dark mode — intentional hero treatment.
