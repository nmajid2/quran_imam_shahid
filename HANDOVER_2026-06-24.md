## 1. What this is (unchanged)
A cross-platform (Android now) **personal Quran companion**: read the Quran, hear
recitation, understand each ayah via **authentic Shia tafsir**, look up **word
meanings (Mufradat)**, and use AI (voice or text) to navigate, search, and ask
questions — futuristic galaxy UI. Owner: **Majid** (single user, not public).
Default language **Persian (fa)**; also en / nl. Arabic is the sacred text.

**Architecture: the app is self-contained.** No gateway. All Quran text, search,
reciter catalog, audio URLs, **tafsir** and **Mufradat lexicon** are bundled in
the APK and served on-device. The **only** network calls go **directly to
OpenAI** — chat (Q&A/locate/route), Whisper STT, and now **TTS** — with the key
baked in at build time via `--dart-define=OPENAI_API_KEY`. Recitation MP3s stream
from the EveryAyah CDN.

Two hard rules still hold: (1) Arabic text is byte-accurate from verified Tanzil,
never AI-generated; (2) the app executes only a **fixed intent allow-list**, never
a model-returned command string.

## 4. What changed this session (deep dive)

### 4a. The Ask flow is now TWO-PART (`features/ai/ask_quran_sheet.dart`)
For a home/surah "Ask the Quran" question (mode = **Answer with sources**, the
default), `_run` produces two parts and **streams** them:

1. **Out-of-tafsir part** (`answerBeyond`) runs **first** — broader, **comparative**
   sources (now explicitly **non-Islamic + academic**: Bible/Tanakh/Talmud,
   historical/orientalist scholarship, AND broader Islamic), each claim attributed,
   precise citations. Shown first (progressive display).
2. **Quran + tafsir part** (`answer`) runs **second** and is given the OOS draft to
   **comment on** (confirm/qualify/correct + add tafsir nuance), grounded ONLY in
   the bundled material. It reports `in_scope`.

Display order: **OOS first, then the tafsir section** (header "بر پایهٔ قرآن و
تفاسیر (در بررسی پاسخ بالا)"). Same 3 model calls (locate → beyond → answer), just
reordered so the tafsir can comment — **no extra round**.

- **Per-question gating** (router flag `tafsir_only`): when the user scopes to the
  Quran/tafsir ("show the Quran ayah that …", "از قرآن", "according to the Quran"),
  the OOS part is **skipped** for that question.
- **Conversation-aware**: follow-ups send the full prior conversation (both parts,
  `AskTurn.fullAnswer`) to `answer`, `answerBeyond`, AND `locateAyat` (so an
  elliptical follow-up's ayat are resolved in context).

### 4b. Always-on Mufradat + word/grammar analysis (`features/ai/openai_client.dart`)
`_attemptSystem` now **always** enriches the tafsir answer with **word-root
(Mufradat) + grammar** analysis of the key words, using it to **validate the
tafsir** (e.g. مسح "wipe" vs غسل "wash"); a full **word-by-word** pass when the
user asks to analyze the words. References must name **only the provided editions**
(al-Mizan/Nemooneh/Noor) + **Mufradat (Raghib)** + ayat — never hallucinate
Majma' al-Bayan / al-Tibyan (that bug made al-Mizan look "missing"). The previous
`word_analysis` mode flag was **removed** (it's always on now).

### 4c. ⚠️ Tafsir excerpts are cleaned before the AI sees them (`features/reader/ai_summary.dart`)
**Important fix.** al-Mizan passages open with the surah header + the full Arabic
of every ayah in the block; the old 1200-char excerpt was therefore **all verse
text, zero commentary**, so the model ignored al-Mizan. `_cleanTafsirForAi` now
strips ```` ```arabic ````/```` ```farsi ```` fences, `<span>` anchors, headings and
rules (the AI already has the Arabic + translation), then truncates to **2500**
chars — so the excerpt is real commentary. Applies to all editions, in the Ask
flow and the per-ayah summary. (Verified the tafsir DBs are all embedded and
correct; al-Mizan was never actually missing — see §6.)

### 4d. Command routing fixes (`features/ai/openai_client.dart` `_commandSystem`, `features/assistant/`)
- **Content-recite**: "read/recite the ayah that says/about X" with no surah →
  `ReciteCommand(surah:null, query:…)`; the sheet/dispatcher **locate the ayah by
  meaning** (`locateAyat`) then recite that one ayah. (`ReciteCommand.surah` is now
  nullable; `app_command.dart`.)
- **Single-ayah recite stops**: "read ayah 5" sets `ayah` (not `from_ayah`) →
  `continuous:false`. Surah / "from ayah X" keep `continuous:true`.
- **search vs ask**: search_quran is now a **literal word/name lookup** only; a
  meaning-based "which ayah says/explains X" routes to **ask**. search query is the
  **bare keyword** (honorifics/filler stripped) so word-AND search matches.
- **Follow-ups in the Ask sheet** route through `routeCommand` too — a follow-up
  that is really a command (recite/open/search) executes (recite plays in the
  background via a `_MiniPlayer`; nav pushes a page over the sheet).

### 4e. Out-of-tafsir modes (`core/config.dart`, settings) — unchanged semantics
`outOfScopeModeProvider`: **with_sources** (default, both parts), **tafsir_only**
(no OOS part), **ask_first** (confirm before OOS). Four AI model **roles** still:
`answerModelProvider` (locate/answer/summarize), `oosAnswerModelProvider`,
`classifyModelProvider`, `refineModelProvider`. Defaults all `gpt-4o-mini` in
config, BUT the device currently has **gpt-4o** selected for Answer/OOS (persisted
in secure storage — survives rebuilds; not in code).

### 4f. Unified AI history (`features/ai/ask_session*.dart`, `ask_sessions_sheet.dart`)
**Every** AI conversation is now saved to the home "Ask AI" history, labeled by
scope:
- `AskSession` gained `surah` / `ayah` / `ayahEnd` + `scopeLabel`. The history list
  shows a badge: none = whole-Quran, "Surah N", or "N:ayah(–end)".
- Surah-scoped "Ask about this surah" (reader FAB) now **persists** (was
  ephemeral). The per-ayah **AI summary sheet** lazily creates a session and saves
  each Q&A turn (`_saveToHistory`). Reopening keeps the surah scope.
- `AskTurn` also gained `beyondAnswer/beyondExample/beyondReferences` (the OOS
  part) and `usage` (token metering) — all persisted.

### 4g. Text-to-speech (NEW) — `features/ai/tts_controller.dart`, `part_audio_player.dart`
- **Engine: OpenAI TTS** (`/v1/audio/speech`, model `gpt-4o-mini-tts`).
  `OpenAiClient.synthesizeSpeech(voice, text, speed)` → MP3 bytes.
- **`TtsController`** (StateNotifier, own just_audio player, separate from
  recitation): a **queue** with `play` (replace) / `enqueue` (after current).
  Clips **synthesize eagerly on creation**, so part 2's audio is prepared **in the
  background while part 1 plays** → no gap. Drain loop advances on true
  end-of-clip (pause doesn't skip). Exposes play/pause/`toggle`, `seekBy`/`seekTo`,
  `setVolume`, position/duration via streams. `ttsIdFor(text)` keys parts.
- **`PartAudioPlayer`** under every answer part: collapsed = "🔊 Listen · ≈$cost
  TTS"; expanded = play/pause, ±10s, scrubber w/ times, volume, stop + a live
  TTS cost line.
- **Auto-read**: only when the question was asked **by voice** (`fromVoice`
  threaded AiInputBar → sheets → home dispatcher) and read-aloud is enabled.
  Plays the OOS part, enqueues the tafsir part.
- **Settings** (`settings_page.dart`): "Read answers aloud" toggle, **speed slider**
  (0.5–2.0×), **narrator picker** (11 voices) each with a **▶ sample preview**, and
  a language note (multilingual; Persian experimental). Persisted via
  `ttsEnabledProvider` / `ttsVoiceProvider` / `ttsSpeedProvider` (seeded in
  `main.dart`).

### 4h. Per-response token usage + cost (NEW) — `features/ai/ai_usage.dart`
- `_chatJson` records each call's `usage` (`prompt_tokens`, `completion_tokens`,
  `prompt_tokens_details.cached_tokens`) into an optional collector forwarded
  through `locateAyat`/`answer`/`answerBeyond`/`routeCommand`/`summarize`.
- The Ask sheet collects usage for the whole question into `AskTurn.usage`
  (persisted) and renders a **"⚡ Tokens & cost"** box: total USD + per-model
  tokens (in/cached/out) and per-model cost.
- Pricing table in `ai_usage.dart` (per 1M tokens, with cached rates):
  gpt-4o-mini 0.15/0.075/0.60, gpt-4o 2.50/1.25/10, gpt-5-mini 0.25/0.025/2,
  gpt-5.4-mini 0.75/0.075/4.5, gpt-5.1 1.25/0.125/10. **Keep in sync with OpenAI.**
- **TTS cost is ESTIMATED** (the speech endpoint returns no token counts):
  input tokens from text (Latin ≈4 ch/tok, Persian/Arabic ≈2), audio-output tokens
  from **real playback duration** (~1250 tok/min; OpenAI's ~$0.015/min) or the 6×
  ratio before duration is known. Rates: $0.60/1M text-in, $12/1M audio-out. Shown
  on the `PartAudioPlayer`. **Not** counted: Whisper STT (per-minute) and the
  initial home routing call (happens before the sheet opens).

### 4i. Network resilience (`features/ai/openai_client.dart`)
`_postWithRetry` wraps every OpenAI POST (chat + TTS): up to **3 attempts** with
backoff on dropped/reset connections, per-attempt **75 s timeout**, and on 429/5xx.
Fixes the intermittent **"Connection reset by peer"** (stale HTTP keep-alive reuse)
that previously showed a manual Retry.

### 4j. Small fixes
- Home smart box now **clears** after submit (`surah_list_page.dart` `_onSubmit`
  resets `_searchCtrl` + `_query`).

## 6. Decisions & caveats (don't re-litigate)
- **Two-part answers, OOS-first, tafsir comments on OOS** — user's explicit choice.
  `tafsir_only` per-question (or the global mode) skips the OOS part.
- **Mufradat/word+grammar analysis is always-on**; the grounded part may cite ONLY
  the 3 embedded tafsirs + Mufradat + ayat.
- **al-Mizan IS embedded** and correct (DB has 58,896 chars for 5:6 etc.). It only
  *looked* missing because (a) the AI hallucinated other tafsir names in citations
  — now constrained, and (b) the un-cleaned excerpt was all verse text — now
  cleaned. Don't "re-add" al-Mizan.
- **OOS sources are comparative** (incl. non-Islamic). The grounded part stays
  strictly Shia tafsir + Mufradat.
- **TTS = OpenAI cloud**, auto-read only for voice questions; per-part manual
  players for everything. Persian TTS is experimental/accented (flagged in UI).
- **TTS cost is an estimate** (no token counts from the endpoint); chat cost is
  exact. Pricing table is hard-coded — update when OpenAI changes prices.
- **Models**: code defaults gpt-4o-mini; the device has gpt-4o selected for
  Answer/OOS (better instruction-following for word analysis; slower/pricier).
- **Network**: transient resets auto-retry (×3). A persistent failure is a real
  outage.
- AI citations are **model-generated, not verified** — a real reference DB is still
  the long-term fix.
